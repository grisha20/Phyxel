# Газовая симуляция

## Область модели

Общий gas solver работает для любого материала `kind: gas`, кроме материалов с флагом `flame`. Он не знает ID steam, smoke, gas или CO₂. Поведение определяется `SimulationKind`, `Density`, `Mass`, температурой и lifetime.

Одна клетка по-прежнему содержит один `MaterialIndex`. Поэтому solver не создаёт химическую смесь: разные газы сохраняют идентичность и могут только переставляться.

## Расписание

`FixedStepGasScheduler` работает на 60 Гц и не зависит от частоты кадров. На каждом tick выполняются четыре in-place dispatch по непересекающимся парам:

1. вертикальные соседние пары одного parity; parity чередуется;
2. горизонтальные соседние пары первого parity;
3. горизонтальные соседние пары второго parity;
4. горизонтальный long-range span, вращающийся по длинам 1/4/16/64 и со сдвигающимся origin.

Так локальное выравнивание остаётся устойчивым, а газ проходит через широкие связанные камеры без фиксированных полос на границах блоков.

## Сохранение величин

При делении/слиянии одинакового газа сохраняются:

- суммарная `Mass`;
- тепловая энергия через mass-weighted temperature и `HeatCapacity`;
- mass-weighted `Lifetime`;
- `MaterialIndex`.

Pressure, `BodyId` и rest state для создаваемых gas portions сбрасываются. `CellMaterials` обновляется синхронно.

Порог `GasMinimumRepresentableMass = 0.01` — минимальная отдельная порция, а не порог удаления. Sub-threshold остаток остаётся существовать. Если обе целевые доли слишком малы, весь пакет перемещается на одну сторону с вероятностью, равной целевой доле; hash детерминирован по клеткам/tick. Это сохраняет массу и устраняет постоянный directional bias.

## Плотность

Два разных газа не объединяются. В вертикальной паре они меняются местами, если верхний плотнее нижнего. Текущие core значения:

| Материал | Density |
|---|---:|
| `core:steam` | 0.03 |
| `core:smoke` | 0.04 |
| `core:gas` | 0.08 |
| `core:co2` | 0.12 |

`core:fire` имеет density 1.0, но флаг `flame` исключает его из этой таблицы поведения. Огонь перемещается и завершается через cellular/combustion/lifecycle passes.

## Пар и температура

Gas redistribution сохраняет тепловую энергию при объединении и делении пара. Отдельно thermal pass применяет `ambientCooling` пара к 20 °C с rate 0.04. После пересечения фазового порога 95 °C общий phase pass превращает пар в воду; вода затем движется обычной liquid-физикой.

## Производительность

Газовый pass использует GPU timestamps и неблокирующий `DoNotFlush`. Последнее измерение на RTX 4070:

| Масштаб | Сетка | Средний gas time |
|---:|---:|---:|
| 25% | 480×270 | 0.149 мс |
| 50% | 960×540 | 0.255 мс |
| 75% | 1440×810 | 0.807 мс |
| 100% | 1920×1080 | 0.804 мс |

## Acceptance-контракт

- `gas_uniform_distribution`: широкое заполнение камеры, отсутствие parity-полос, препятствия, точная масса и density ordering steam/smoke/gas/CO₂ при 30/60/100/144 FPS.
- `steam_distribution_and_cooling`: подъём, широкое распределение, ambient cooling, конденсация и сохранение mass при 30/60/100 FPS.
- `thermal_gas`: mass-weighted temperature/energy одинакового газа и запрет смешения разных `MaterialIndex`.
- `external_gas`: внешний JSON-газ получает те же возможности без ID-specific кода.
- `combustion_chain`: flame остаётся отдельным transient-материалом, создаёт smoke и сохраняет валидный lifetime.

## Известная граница

Модель не хранит доли компонентов. Для настоящей смеси воздуха, дыма, пара и CO₂ потребуется отдельный compact composition storage и новые правила сохранения массы/энергии; текущий `MaterialIndex` для этого недостаточен.
