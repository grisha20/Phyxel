# Phyxel UI Design Specifications

Этот документ фиксирует визуальную и функциональную концепцию нового интерфейса Phyxel, основанную на анализе The Powder Toy (TPT), концепт-макета Phyxel и идей Algodoo.

---

## 1. Принципы, заимствованные из The Powder Toy

1. **Категориальная нижняя панель материалов**:
   - Быстрый выбор элементов по категориям (Порошки, Жидкости, Газы, Твёрдые, Горение, Инструменты) вместо единого длинного списка.
   - Фильтрация материалов по вкладкам и возможность горизонтального выбора.
2. **Информативная строка состояния (Status Bar)**:
   - Отображение свойств ячейки под курсором (название материала, температура) и динамических параметров симуляции (FPS, активные клетки).
3. **Блокировка взаимодействия с миром над UI (`PointerConsumed`)**:
   - Курсор мыши поверх UI-панелей полностью блокирует вызовы кисти и спавн элементов в мире.
4. **Единая кнопка Паузы/Продолжения**:
   - Переключение режима симуляции в один клик с четким изменением текста и цвета состояния.

---

## 2. Элементы из приложенного макета и Algodoo

1. **Современная тёмная тема (Dark Graphite Editor)**:
   - Использование графитового фона (`#0D0F12`), чуть более светлых панелей (`#16191E`), тонких рамок (`#2C313B`) и аккуратных скруглений.
2. **Левая вертикальная панель инструментов**:
   - Постоянно доступные инструменты (Кисть, Ластик, Температура, Панорама) с наглядными собранными line-art иконками.
   - Слоты под продвинутые фигуры и инструменты (Линия, Прямоугольник, Круг, Заливка, Пипетка), помеченные подсказкой «Скоро» для нереализованных в V1 функций.
3. **Правая панель контекстных свойств (Properties Panel)**:
   - Настройка размера кисти, плотности спавна, целевой температуры и параметров мира (гравитация, гидравлика) в отдельной выделенной области справа.
4. **Верхняя панель управления (Top Bar)**:
   - Геометрический логотип Phyxel, быстрые действия (Сохранить, Загрузить, Пауза, Настройки) и метрики производительности.

---

## 3. Отличия и решения, разработанные специально для Phyxel

1. **GPU-ориентированная адаптивная сетка**:
   - Центральный viewport симуляции адаптируется под любые разрешения (1280×720, 1600×900, 1920×1080, 2560×1440) и DPI масштабы Windows (100%, 125%, 150%) без перекрытия панелями.
2. **Динамическая категория JSON-материалов**:
   - Автоматическая группировка материалов по их базовым свойствам (`kind`, `flags`) с поддержкой опционального `ui.category` поля без жесткой привязки к `RuntimeIndex`.
3. **Модульная C# архитектура UI**:
   - Вынос логики из главного класса `PhyxelGame.cs` в специализированные компоненты: `UiLayoutCalculator`, `UiTheme`, `UiIconRenderer`, `UiCategoryPalette`, `UiPropertiesPanel`, `UiTopBar`, `UiStatusBar`.
4. **Отсутствие мусора за кадр (Zero Garbage Allocations)**:
   - Кэширование всех прямоугольников, иконок, сформированных текстов и списков видимых карточек.
